# skills_inbox — ResearchForge Collection Inbox

Throw skills you want to integrate into ResearchForge into **this folder**. I (Claude) will pick them up during auto-polling, **no need to find me each time**.

## How to Ingest
- One skill = one folder (containing `SKILL.md` + optional references/scripts), drag the whole thing in;
- Or single `.md` file, `.zip` archive, also OK;
- **Papers also accepted**: place PDF here, or write DOI/links in `_papers.txt` (one per line) — I'll extract its method/data prerequisite/analysis register in catalog (can also locate replication code side-by-side);
- Or if you just want to give me an existing path, write the path in `_paths.txt` (one per line) also works.

## What I'll Do (read-only / won't touch your originals)
1. Read each skill, extract: **covered discipline, doable analyses/models, required data prerequisites, outputs**;
2. Register in `../docs/analysis-catalog/` (layer ② "analysis catalog" draft);
3. After processing move originals to `_processed/` and mark source in catalog.

> Note: Currently in design/collecting phase, I won't touch product code. Catalog is just knowledge base draft.
