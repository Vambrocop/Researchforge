# Meta 分析（Meta-analysis）— ResearchForge 使用与协作说明

> 给协作者看的一页纸：怎么把**多个人、多篇研究**的结果**合并**成一个总效应，怎么准备数据、怎么跑、结果怎么读。
> 引擎实现：R `metafor`（领域金标准），随机效应 REML 为默认。

---

## 1. 一句话原理

Meta 分析 = 把**若干独立研究**各自的「效应量」按其精度（方差/样本量）**加权合并**，得到一个总效应 + 置信区间，并量化研究之间的**异质性**。
**每一行 = 一篇研究（或一个对比）**。所以多人协作很自然：大家按同一张表的列各填各的研究行，最后把行**拼起来**就能一起跑——这就是「合并其他人的」。

---

## 2. 数据格式（三选一，列名不区分大小写；都可用 `--config` 显式指定）

### 格式 A — 已算好的效应量（最通用，推荐多人协作用这个）
| 列 | 含义 | 必需 |
|---|---|---|
| `study` | 研究标签（作者+年份等） | 可选（强烈建议，森林图用它） |
| `yi` | 效应量（如 log 反应比 lnRR、标准化均差 SMD、log(OR)…） | ✅ |
| `vi` 或 `sei` | `yi` 的**抽样方差** `vi`，或**标准误** `sei`（引擎自动 `vi=sei²`） | ✅（二选一） |

> 适合：每个人已经各自算好效应量。大家只要保证 `yi` 用的是**同一种效应量度量**（都用 lnRR，或都用 SMD），就能合并。

### 格式 B — 两组原始均值（→ 自动算标准化均差 SMD）
| 列 | 含义 |
|---|---|
| `study` | 研究标签（可选） |
| `m1, sd1, n1` | 处理组 均值 / 标准差 / 样本量 |
| `m2, sd2, n2` | 对照组 均值 / 标准差 / 样本量 |

> 适合：农学/生态最常见——「处理 vs 对照」的均值±标准差。引擎用 `escalc(measure="SMD")` 自动换算。
> （想要原始均差而非标准化，`--config '{"measure":"MD"}'`。）

### 格式 C — 2×2 计数（→ 自动算比值比 OR）
| 列 | 含义 |
|---|---|
| `study` | 研究标签（可选） |
| `ai, bi` | 组1 的 事件数 / 非事件数 |
| `ci, di` | 组2 的 事件数 / 非事件数 |

> 适合：二分类结局（发病/未发病等）。默认 OR，可 `--config '{"measure":"RR"}'`(风险比) 或 `"RD"`(风险差)。

---

## 3. 多人合并的标准流程（重点）

1. **约定一张模板表**：选定上面**某一种格式**，把列名固定下来发给每个人（例如格式 A 的 `study,yi,sei`）。
2. **各自填行**：每人把自己负责的研究填进**相同的列**，一行一篇研究。
3. **拼表**：把所有人的 CSV 行直接**纵向拼接**（同样的表头，行接行）成一个 `studies.csv`。
   - 关键：所有人 `yi` 必须是**同一种效应量度量**、方向一致（例如「正值=处理更好」）。
   - 缺失值的行会被自动剔除（并在结果里体现于有效研究数 k）。
4. **跑**：见第 4 节。森林图里每篇研究一行，最底是合并总效应。

> 这样无论几个人贡献，合并都是「把行拼起来」，不需要谁懂代码。

---

## 4. 怎么跑

```bash
# 默认随机效应 (REML)，自动识别列名与格式
py -3 -m researchforge.cli run studies.csv meta_analysis

# 显式指定列名 / 度量 / 模型（任意组合）
py -3 -m researchforge.cli run studies.csv meta_analysis --config '{"effect":"lnRR","se":"se_lnRR","study":"ref","measure":"GEN","method":"REML"}'
```

**config 键**：`effect`(yi 列)、`variance`(vi 列)、`se`(sei 列)、`study`(标签列)、`measure`（`GEN`/`SMD`/`MD`/`OR`/`RR`/`RD`）、`method`（`REML` 默认随机效应 / `DL` / `FE` 固定效应）。
不传则按列名自动识别。

---

## 5. 产出

- `meta_summary.txt` — 合并效应 + 95% CI + p、异质性（I²/τ²/Q 检验）、解读。
- `study_effects.csv` — 每篇研究换算后的 `yi`/`vi`（便于核对/再利用）。
- `forest.png` — **森林图**：每篇研究效应±CI + 底部合并总效应。
- `funnel.png` + **Egger 检验** — 漏斗图对称性 → 发表偏倚信号。

---

## 6. 结果怎么读

- **合并效应 + 95% CI**：CI 不跨 0（或 OR/RR 不跨 1）即统计显著。OR/RR 在 log 尺度合并、报告时已 `exp()` 回原尺度。
- **I²（异质性）**：研究间差异占比。`<25%` 极低、`25–50%` 低、`50–75%` 中、`>75%` 高。
  **I²>50% 要当心**：一个「平均」总效应可能掩盖了不同研究的真实差异 → 应找**调节变量**（亚组/meta 回归，后续可加）。
- **τ²**：研究间真实效应的方差（随机效应假定各研究真效应不同）。
- **Egger 检验 p<0.05**：漏斗图不对称，**可能**有发表偏倚（也可能是真实的小样本效应，需结合判断）。

## 7. 必读的坑（诚实披露）

- **垃圾进垃圾出**：meta 分析会继承被合并研究的偏倚，合并**不能**修正原始研究设计的问题。
- 度量必须统一、方向一致，否则合并无意义。
- 研究数太少（k<~5）时异质性估计与 Egger 检验都不稳。
- 当前实现：合并 + 异质性 + 森林/漏斗 + Egger。**亚组/meta 回归（调节变量）、trim-and-fill 补缺** 为后续增量。
